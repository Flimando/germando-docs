Erste Schritte

Pelican Panel ist dafür ausgelegt, auf Ihrem Webserver zu laufen.

Sie sollten unsere Dokumentation durchlesen. Wir haben viel Zeit damit verbracht, diese Dokumentation für die Community zu erstellen. Bitte nehmen Sie sich etwas Zeit, sie zu lesen, bevor Sie in den Foren um Hilfe bitten!

Sie sollten grundlegende Kenntnisse von Linux haben, bevor Sie fortfahren!
Pelican befindet sich derzeit in der Beta-Phase! Einige Dinge können sich zwischen Beta-Versionen ändern oder brechen!

Wahl eines Betriebssystems (OS)

Pelican läuft auf einer Vielzahl von Betriebssystemen, wählen Sie also das aus, mit dem Sie am vertrautesten sind. Hinweis: Andere Betriebssysteme, die unten nicht aufgeführt sind, könnten immer noch funktionieren.

OpenVZ, außer bei spezieller Konfiguration, funktioniert nicht mit Pelican.
Betriebssystem Version Unterstützt Hinweise
Ubuntu 20.04 ⚠️ Keine SQLite-Unterstützung, Ubuntu 20.04 EoL ist April 2025, nicht empfohlen
22.04 ✅
24.04 ✅ Dokumentation basiert auf Ubuntu 24.04 als Basis-OS.
Rocky Linux 9 ✅
Debian 11 ⚠️ Keine SQLite-Unterstützung
12 ✅

SQLite-Unterstützung hängt davon ab, dass libsqlite3-0_3.35+ auf dem Host-System vorhanden ist. Ubuntu 20.04 & Debian 11 erfüllen diese Anforderung nicht.

Abhängigkeiten

Für das Panel müssen Sie PHP 8.4 (empfohlen), 8.3 oder 8.2 mit den folgenden Erweiterungen installieren:
gd, mysql, mbstring, bcmath, xml, curl, zip, intl, sqlite3 und fpm.

Sie benötigen auch einen Webserver. Derzeit werden Apache, NGINX oder Caddy unterstützt.

Wenn Sie MySQL oder MariaDB für die Panel-Datenbank verwenden möchten, stellen Sie sicher, dass Sie entweder MySQL 8+ oder MariaDB 10.6+ installieren. (sowohl Client als auch Server!)

Schließlich benötigen Sie für einige Befehle während der Installation curl, tar und unzip.

Bitte stellen Sie sicher, dass Sie alle benötigten Abhängigkeiten installiert haben, bevor Sie fortfahren!

Verzeichnisse erstellen & Dateien herunterladen

Der erste Schritt in diesem Prozess ist die Erstellung des Ordners, in dem das Panel leben wird, und dann in diesen neu erstellten Ordner zu wechseln.

sudo mkdir -p /var/www/pelican
cd /var/www/pelican

Sobald Sie ein neues Verzeichnis erstellt und in dieses gewechselt haben, müssen Sie die Panel-Dateien herunterladen. Dies ist so einfach wie die Verwendung von curl zum Herunterladen der neuesten Version.

curl -L https://github.com/flimanda/germando-panel/releases/latest/download/panel.tar.gz | sudo tar -xzv

Composer installieren

Als nächstes richten wir Composer zusammen mit den erforderlichen Abhängigkeiten ein.

curl -sS https://getcomposer.org/installer | sudo php -- --install-dir=/usr/local/bin --filename=composer
sudo COMPOSER_ALLOW_SUPERUSER=1 composer install --no-dev --optimize-autoloader
Auch wenn Composer Ihnen mitteilt, dass Sie veraltete Abhängigkeiten haben, führen Sie nicht composer update aus!
